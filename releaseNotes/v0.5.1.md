# Release Notes v0.5.1

## What's New

### Improvements

- **Simplified executor API**: Refactored `InterationExecutor` to accept `IWebAccessor` directly via constructor injection, removing internal dependency on `IServiceProvider`
- **Renamed HTTP accessor**: Renamed `HttpWebAccessor` to `HttpClientWebAccessor` for clarity and consistency with the underlying `HttpClient` implementation
- **Enhanced error debugging**: When output validation fails, the response HTML is now included in the outputs for easier debugging
- **Null safety**: Added null check in `GetXpathNodeValue` to prevent `NullReferenceException` when XPath selects a non-existent node
- **Comprehensive documentation**: Completely revamped README with detailed usage examples, script reference, and API documentation

## Breaking Changes

- **Constructor change**: `InterationExecutor` no longer has a parameterless constructor. You must now pass an `IWebAccessor` instance:
  ```csharp
  // Before (v0.5.0)
  var executor = new InterationExecutor();
  
  // After (v0.5.1)
  var executor = new InterationExecutor(new HttpClientWebAccessor());
  ```

- **Class renamed**: `HttpWebAccessor` has been renamed to `HttpClientWebAccessor`

## Bug Fixes

- Fixed potential `NullReferenceException` when XPath expressions return null nodes

## Dependencies

- HtmlAgilityPack 1.12.4
- Microsoft.Extensions.DependencyInjection 9.0.0
